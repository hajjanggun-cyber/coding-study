## 🔧 주요 기능

### 1. 6개 카테고리 관리
- **Coding용어**: 코딩 관련 용어
- **프롬프트**: AI 프롬프트
- **참고URL**: 웹사이트 링크
- **아이디어**: 프로젝트 아이디어
- **실전과정**: 프로젝트 진행 과정
- **강의정리**: 강의 내용 정리

### 2. CRUD 기능
- ✅ **등록**: 새 데이터 추가
- ✅ **조회**: 목록 보기 (최신순/가나다순)
- ✅ **수정**: 인라인 수정 (카드에서 바로 수정)
- ✅ **삭제**: 선택 삭제 (여러 개 동시 삭제 가능)
- ✅ **이동**: 카테고리 간 이동

### 3. UI/UX 특징
- **인라인 수정**: 수정 버튼 클릭 시 바로 편집 가능
- **자동 높이 조절**: textarea가 내용에 맞춰 자동 확장
- **접기/펼치기**: 실전과정/강의정리에서 긴 내용 접기 가능
- **URL 자동 링크**: URL을 자동으로 클릭 가능한 링크로 변환
- **반응형**: 모바일/PC 모두 지원

### 4. 성능 최적화
- **캐싱**: 모든 데이터를 메모리에 캐싱하여 빠른 로딩
- **낙관적 UI**: 서버 응답 기다리지 않고 즉시 화면 갱신
- **백그라운드 동기화**: 사용자는 UI 사용, 뒤에서 서버 저장

## 🐛 주요 이슈 및 해결

### 이슈 1: 수정 모드에서 제목이 잘림
**해결**: `.card.editing .card-term`의 너비를 `auto`로 변경

### 이슈 2: textarea 높이가 자동으로 늘어나지 않음
**해결**: 렌더링 후 50ms 지연 후 높이 자동 조절

### 이슈 3: 수정 모드에서 내용 입력창이 좁게 표시됨
**해결**: 수정 모드일 때 `.card-content`를 세로 배치(`flex-direction: column`)로 변경

### 이슈 4: 데이터 저장/수정 직후 목록이 사라지거나 초기화됨
**해결**: 저장/수정/삭제 후 `fetchAllData()` 자동 호출 로직 제거

**브라우저 캐시 이슈**: CSS 변경 후 강력 새로고침 필요 (`Ctrl + Shift + R`)
